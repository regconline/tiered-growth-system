# 🔐 1. CONTACT FORM SECURITY (CRITICAL)

Implement enterprise-level security for the contact form:

### CSRF Protection

* Add **CSRF tokens** to all form submissions.
* Validate CSRF tokens server-side.
* Reject requests without valid tokens.

### Input Validation & Sanitization

* Validate all fields (name, email, message, etc.).
* Sanitize inputs to prevent:

  * XSS (Cross-Site Scripting)
  * Injection attacks
* Use strict validation rules:

  * Email must be properly formatted
  * Message length limits (e.g. max 1000 chars)

### Rate Limiting

* Prevent spam by limiting submissions:

  * Example: max 5 submissions per IP per 10 minutes

### Bot Protection

* Add one of:

  * Honeypot field (hidden input)
  * OR Google reCAPTCHA (v2 or v3 preferred)

### Secure Headers

Add HTTP security headers:

* `Content-Security-Policy`
* `X-Frame-Options: DENY`
* `X-Content-Type-Options: nosniff`
* `Referrer-Policy`
* `Strict-Transport-Security`

### HTTPS Enforcement

* Force HTTPS redirects
